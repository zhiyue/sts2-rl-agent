# Bridge Mod Build Guide

Step-by-step guide to building and installing the STS2 RL Bridge Mod. This mod connects the trained Python RL agent to the real Slay the Spire 2 game via TCP.

This guide covers the current `bridge_mod/` implementation, which is based on the game's built-in AutoSlay automation framework.

---

## Prerequisites

### 1. .NET 9 SDK

The game and mod template target .NET 9.0.

**Windows (PowerShell):**
```powershell
Invoke-WebRequest -Uri https://dot.net/v1/dotnet-install.ps1 -OutFile dotnet-install.ps1
./dotnet-install.ps1 -Channel 9.0
```

**Linux/macOS:**
```bash
curl -sSL https://dot.net/v1/dotnet-install.sh | bash /dev/stdin --channel 9.0
```

Verify:
```bash
dotnet --version
# Should show 9.x.x
```

### 2. Godot 4.5.1 Mono

Required only for `dotnet publish` (PCK generation). Not needed for `dotnet build`.

1. Download from: https://github.com/godotengine/godot/releases/tag/4.5.1-stable
2. Get the **Mono** variant for your platform:
   - Windows: `Godot_v4.5.1-stable_mono_win64.zip`
   - Linux: `Godot_v4.5.1-stable_mono_linux_x86_64.zip`
   - macOS: `Godot_v4.5.1-stable_mono_macos.universal.zip`
3. Extract to a known location:
   - Windows default: `C:/megadot/Godot_v4.5.1-stable_mono_win64/`
   - Linux/macOS: adjust `GodotPath` in the `.csproj`

The `.csproj` file has a `GodotPath` property that must point to the Godot editor binary. The default Windows path is pre-configured.

### 3. Slay the Spire 2

Install via Steam. The build system auto-detects the Steam library path on all platforms:

- **Windows:** Reads from registry, falls back to `C:/Program Files (x86)/Steam/steamapps`
- **Linux:** `~/.local/share/Steam/steamapps`
- **macOS:** `~/Library/Application Support/Steam/steamapps`

If STS2 is installed in a non-default Steam library, set `SteamLibraryPath` in the `.csproj` or via MSBuild property.

### 4. Mod Template (optional, for new projects)

If starting a new mod from scratch (not needed for this project since it already exists):

```bash
dotnet new install Alchyr.Sts2.Templates
dotnet new alchyrsts2mod -n MyModName --ModAuthor "yourname"
```

## Building

### Build Only (DLL)

From the `bridge_mod/` directory:

```bash
cd bridge_mod
dotnet build
```

This:
- Compiles all `.cs` files into `STS2BridgeMod.dll`
- Copies the DLL and `STS2BridgeMod.json` to the game's mods folder
- Copies BaseLib (dependency) to the mods folder

Output:
```
Slay the Spire 2/
  mods/
    BaseLib/
      BaseLib.dll
      BaseLib.pck
      BaseLib.json
    STS2BridgeMod/
      STS2BridgeMod.dll
      STS2BridgeMod.json
```

### Publish (DLL + PCK)

```bash
cd bridge_mod
dotnet publish
```

This does everything `dotnet build` does, plus:
- Invokes Godot to export the PCK file (contains `mod_manifest.json` and any Godot resources)
- Copies the PCK to the mods folder

Output:
```
Slay the Spire 2/
  mods/
    STS2BridgeMod/
      STS2BridgeMod.dll
      STS2BridgeMod.pck      <-- Generated by Godot export
      STS2BridgeMod.json
```

## Mod Files

| File | Purpose |
|------|---------|
| `MainFile.cs` | Entry point. Applies Harmony patches, starts TCP server, launches AutoSlayer. |
| `BridgeServer.cs` | TCP server (port 9002). Handles JSON message I/O with the Python agent. |
| `RlAutoSlayer.cs` | Replaces AutoSlay's random handlers with RL agent decision handlers. |
| `RlCombatHandler.cs` | Serializes combat state to JSON, receives card-play/end-turn actions. |
| `RlMapHandler.cs` | Serializes map state, receives node selection. |
| `RlCardSelector.cs` | Handles card selection screens (rewards, upgrades, transforms). |
| `RlCardRewardScreenHandler.cs` | Card reward screen handler. |
| `STS2BridgeMod.csproj` | Build configuration with platform detection and auto-copy targets. |
| `mod_manifest.json` | Mod metadata for the STS2 mod loader. |
| `project.godot` | Godot project file (required for PCK export). |

## Verifying the Installation

1. **Start the game** via Steam.

2. **Check the title screen.** If mods loaded correctly, you will see "Running Modded" in the corner.

3. **Check game logs.** Look for:
   ```
   [STS2Bridge] === STS2 RL Bridge Mod Initializing ===
   [STS2Bridge]   Patched: IsReleaseGamePatch
   [STS2Bridge]   Patched: WaitSpeedPatch
   [STS2Bridge]   Patched: AnimationSpeedPatch
   [STS2Bridge] Harmony: 3/3 patches applied.
   [STS2Bridge] TCP server started on port 9002.
   [STS2Bridge] === STS2 RL Bridge Mod Ready ===
   ```

   Log file location:
   - **Windows:** `%APPDATA%/Godot/app_userdata/Slay the Spire 2/logs/godot.log`
   - **Linux:** `~/.local/share/godot/app_userdata/Slay the Spire 2/logs/godot.log`

4. **Test TCP connection.** In a separate terminal:
   ```bash
   python -c "
   import socket, json
   s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
   s.connect(('127.0.0.1', 9002))
   s.sendall(json.dumps({'action': 'ping'}).encode() + b'\n')
   print('Connected!')
   s.close()
   "
   ```

## Troubleshooting Build Errors

### "Slay the Spire 2 data not found"

The build cannot find the game's DLLs (sts2.dll, 0Harmony.dll, GodotSharp.dll).

Fix: Set the correct Steam library path:

```bash
dotnet build -p:SteamLibraryPath="D:/SteamLibrary/steamapps"
```

Or edit `STS2BridgeMod.csproj` and adjust the `SteamLibraryPath` property.

### NuGet restore fails for Alchyr.Sts2.BaseLib

The BaseLib package is hosted on a custom NuGet feed. Check `nuget.config` in the project:

```xml
<configuration>
  <packageSources>
    <add key="sts2-mods" value="https://nuget.pkg.github.com/Alchyr/index.json" />
  </packageSources>
</configuration>
```

If authentication is required, configure a GitHub personal access token:

```bash
dotnet nuget add source https://nuget.pkg.github.com/Alchyr/index.json \
    --name sts2-mods --username YOUR_GITHUB_USERNAME --password YOUR_PAT
```

### Godot PCK export fails

If `dotnet publish` fails at the Godot export step, verify:

1. `GodotPath` points to the correct Godot binary
2. The Godot version is exactly 4.5.1 (not newer)
3. `export_presets.cfg` exists in the project root

If you cannot get PCK export working, `dotnet build` alone is sufficient -- you can manually create a minimal PCK or copy one from a working build.

### "Type or namespace 'MegaCrit' could not be found"

The `sts2.dll` reference is not resolving. Verify:

1. The game is installed and the `data_sts2_windows_x86_64/` (or platform equivalent) directory exists
2. The `sts2.dll` file is present in that directory
3. The `SteamLibraryPath` property resolves correctly

## Speed Patches

The mod includes three Harmony patches that accelerate the game for faster agent evaluation:

| Patch | Target | Default | Effect |
|-------|--------|---------|--------|
| `WaitSpeedPatch` | `Cmd.CustomScaledWait` | 0.1x | Reduces all timed delays by 10x |
| `AnimationSpeedPatch` | `MegaAnimationState.SetTimeScale` | 5.0x | Speeds up Spine animations by 5x |
| `IsReleaseGamePatch` | `NGame.IsReleaseGame` | false | Unlocks AutoSlay and debug features |

Combined, these patches achieve approximately 5-10x faster gameplay compared to normal speed.

## Architecture Note

**bridge_mod/**:
- Patches `NGame.IsReleaseGame()` to unlock the built-in `AutoSlay` system
- Replaces AutoSlay's random decision handlers with RL agent handlers
- AutoSlay handles all UI navigation, screen transitions, and error recovery
- ~200-300 lines of custom handler code (AutoSlay does the heavy lifting)
- More robust against game updates since it uses the game's own automation framework

The v2 approach was adopted after discovering that STS2 ships with a complete `AutoSlay` system (in `MegaCrit.Sts2.Core.AutoSlay`) -- a fully functional auto-play framework that is gated behind `IsReleaseGame()`. By patching that check, we get a mature automation backbone for free.
